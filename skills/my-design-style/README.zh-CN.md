# my-design-style 中文使用指南

`my-design-style` 是一个面向 PPT、网页、App、Dashboard、静态视觉稿和设计模板的视觉风格 Skill。它的重点不是“随便套一个配色”，而是让你和 LLM 先把目标、受众、媒介、风格、素材边界和预览方案说清楚，再把内容转换成稳定、可复用、可维护的设计方案。

这份文档面向中国中文用户：帮助你知道什么时候使用这个 Skill、怎么描述需求、如何选择现有风格、什么时候要求预览，以及如何给出创作空间、素材边界和质量期望。

## 适合什么任务

适合使用 `my-design-style` 的任务包括：

- **PPT / Slides**：课题汇报、答辩、商业汇报、课程展示、政策或文化主题演示。
- **Web 页面**：官网首页、活动落地页、产品介绍页、专题页、内容型页面。
- **App / 小程序界面**：首页、列表页、详情页、表单页、个人中心、运营活动页。
- **Dashboard / 数据看板**：金融数据、科研数据、业务指标、项目进度、管理驾驶舱。
- **静态视觉稿**：海报、封面、KV、社媒长图、展板、视觉方向稿。
- **设计模板**：可反复套用的版式、组件、色彩和视觉系统建议。

如果你只想“一次性生成一张图片”，且不关心设计系统、版式规则或后续复用，通常不需要使用这个 Skill。

## 使用前先准备这些信息

你不必一次性写得很完整，但信息越清楚，LLM 越不需要反复追问。

| 信息 | 你可以这样说 |
| --- | --- |
| 目标产物 | “我要做一页 PPT 封面”、“我要做一个移动端活动页”、“我要设计一个金融 Dashboard”。 |
| 使用场景 | “研究生组会汇报”、“投资人路演”、“校内项目展示”、“文博活动宣传”。 |
| 受众 | “导师和课题组”、“高校学生”、“普通公众”、“金融客户”、“企业管理层”。 |
| 风格 | 直接写 style id，或描述“高校学术感”“中式金融可信感”“新中式文化感”。 |
| 内容材料 | 提供标题、文案、数据、图表、页面结构或已有草稿。 |
| 约束条件 | 尺寸、平台、是否需要响应式、是否必须使用某些 logo / 图片 / 色彩。 |
| 预览要求 | “先给 2 个方向，不要直接做完整稿”、“先出一页风格预览给我确认”。 |
| 创作边界 | “可以更大胆，但保持可读”、“人民币风红金作为价值信号”、“校徽比例要正确”。 |

## 可用风格速查

当前可选的主要风格如下。建议优先使用明确的 style id，这样可以减少歧义。

| 风格 id | 适合场景 | 一句话理解 | 常见中文说法 |
| --- | --- | --- | --- |
| `seu_design_style` | 高校、科研、答辩、课题组、校园品牌、学术网页 | 东南大学 / 高校机构感，偏克制、清晰、结构化 | “东南大学风格”、“高校学术感”、“科研汇报感”、“校级项目展示”。 |
| `renminbi_color_style` | 金融科技、商业报告、支付、资产、价值感、可信品牌 | 借鉴人民币色彩、纸感和细线层次的金融视觉语言 | “中式金融视觉”、“人民币配色”、“可信金融感”、“纸感价值感”。 |
| `chinese_traditional_color_style` | 文博、国风、传统文化、诗词、茶、工艺、文化活动 | 使用中国传统色和可伸缩的现代文化构图，可克制也可仪式化 | “新中式”、“国风但现代”、“博物馆感”、“传统色配色”。 |

如果你不确定选哪个，可以让 LLM 根据用途推荐，但请要求它说明理由，并且只选择一个主风格。

## 推荐沟通流程

### 1. 先说清楚任务

先说明你要做什么，而不是只说“帮我美化一下”。例如：

```text
请使用 my-design-style，帮我把这份研究汇报改成更适合组会展示的 PPT 视觉方案。
目标受众是导师和课题组成员，内容偏科研进展，希望清晰、克制、有高校学术感。
```

### 2. 再确定风格

如果你知道风格，直接点名：

```text
请使用 seu_design_style。
```

如果你不知道风格，让 LLM 推荐：

```text
我不确定该用哪个 style。请根据“金融科技产品首页、面向企业客户、需要可信但不保守”的目标，在现有风格中推荐一个主风格，并说明不选其他风格的原因。
```

### 3. 重要任务先要预览

公开发布、品牌敏感、汇报重要、你自己也不确定方向时，先要求预览：

```text
请先不要生成完整页面。先给我 2 个风格预览方向，每个方向说明色彩、版式、纹理、标题处理和适合/不适合之处。等我确认后再继续。
```

### 4. 确认后再进入完整产出

确认风格方向后，再要求完整设计或代码：

```text
采用第 2 个预览方向。请继续生成完整的页面结构、配色 token、组件样式和首屏 HTML/CSS 实现建议。
```

## 常用提示词模板

### 模板 A：PPT 汇报美化

```text

目标媒介：PPT
使用场景：研究生组会 / 答辩 / 项目汇报
目标受众：导师、评委、课题组成员
推荐风格：seu_design_style
希望效果：学术、清晰、有结构感；封面和章节页可以更有仪式感
输出内容：请给出页面结构、标题层级、配色、字体建议、图表/表格处理方式、每页视觉锚点、跨页视觉节奏，以及每页的版式说明

[粘贴你的内容]
```

```text

推荐风格：renminbi_color_style
希望效果：可信、稳重、有价值感，有中式金融文化气质；红金可以作为价值信号，也可以使用人民币配色、纸感和细线层次

```

## 和 LLM 协商时重点确认什么

当 LLM 追问或给出预览时，你可以重点看这些点：

1. **媒介是否正确**：PPT、网页、App、Dashboard、海报的版式逻辑不同，不要混用。
2. **主风格是否唯一**：默认只选一个主风格，不要把三个风格平均混合。
3. **内容是否优先**：风格应该服务标题、论点、数据和行动路径，而不是盖过内容。
4. **视觉节奏是否规划过**：多页 PPT、长页面或多屏流程应说明每页/每屏的视觉锚点、页面 archetype、motif 轮换和变化规则。
5. **预览是否可执行**：好的预览应说明色彩、版式、纹理、图形、组件、视觉锚点和适用风险。
6. **素材是否丰富**：logo、校徽、纹样、图片、纸币元素等可以来自运行时素材、联网下载、生成素材或用户文件，记录来源/路径方便复用即可。
7. **可读性是否达标**：国风、纸感、纹理、细线不能牺牲文字、图表和按钮识别。


## 保持多页创作变化

生成 PPT、长页面或多屏流程时，除了“统一风格”，也请让 LLM 先给出一个视觉节奏计划：

- 每页/每屏属于什么 archetype，例如封面、目录、章节页、数据页、对比页、流程页、结尾页。
- 每页/每屏至少有什么正文之外的视觉锚点，例如图表、流程图、色卡、边栏、标签框、编号块、图片、低密度纹理或 code-native 几何。
- motif、纹理和布局密度如何轮换，让连续页面有可解释的变化。
- 没有可用图片资产时，如何用图解、色块、swatch、细线、表格和 typography fallback，而不是留下空白或堆砌无关装饰。
- 最后做 variation self-check：是否存在连续版式重复、视觉锚点缺失、装饰无信息角色或可读性下降。

## 风格混用规则

一般情况下，请让 LLM 选择**一个主风格**。如果确实想混合，请明确“主风格”和“次要影响”：

```text
主风格使用 chinese_traditional_color_style，但希望在局部金融数据卡片中吸收 renminbi_color_style 的稳重纸感。以传统色系统作为主风格。
```

不推荐这样说：

```text
把东南大学风、人民币风、国风都融合一下，越高级越好。
```

这种说法会导致风格边界模糊，也容易让 LLM 产生拼贴感。

## 预览什么时候必要

建议在这些情况下要求预览：

- 重要汇报、品牌官网、招生宣传、商业提案等你希望先确认方向的产物。
- 答辩、路演、评审、竞赛等高风险场景。
- 你不确定应该选哪个风格。
- 你希望比较两个方向，例如“更学术”与“更品牌化”。
- 涉及校徽、机构视觉、金融元素、传统文化符号等你想先确认效果的语义。

可以直接说：

```text
这是公开使用的页面，请先做预览协商。不要直接进入完整设计。
```

## 素材边界

使用这个 Skill 时，请注意：

- 每个具体风格都必须有 `assets/<style_name>/` 和 `ASSET_MANIFEST.md`；目录可以为空，空目录表示运行时下载或生成素材，而不是不使用素材。
- `seu_design_style` 使用运行时 `AssetPolicy` 暴露的东南大学相关素材角色，并保持 logo、字标、纹样等比例正确。
- `renminbi_color_style` 借鉴人民币的色彩、层次、纸感和细线语言；素材可以来自运行时策略、代码几何、生成向量、用户素材、联网下载的图片，以及人民币配色、面额色条和货币元素。
- `chinese_traditional_color_style` 主要依赖传统色和现代构图；素材可以来自运行时策略、代码几何、生成向量、用户素材、联网下载的图片，以及祥云、印章、器物等文化元素。
- 让 LLM 通过 `AssetPolicy` 和 manifest 理解素材边界；不同风格的素材可以自由组合使用。
- 如果你提供外部 logo、照片或品牌图，可以同时说明是否必须使用。
- 联网检索和下载素材是整个 Skill 的主动设计能力，应优先使用，而不是只在运行时素材不足时才用。只要环境允许且用户没有禁止联网，就应围绕当前主题搜索相关素材或视觉参考，并与运行时素材、生成向量、代码几何和用户素材比较，选择最能保持风格、贴合语义并增加变化的方案。推荐来源在顶层 `SKILL.md` 中维护；当前使用 Pixabay（`https://pixabay.com/`）和 Iconfont / 阿里巴巴矢量图标库（`https://www.iconfont.cn/`）。可复用素材可放入对应 style 的 `assets/` 边界，记录来源链接和用途即可，方便日后再次查找。

## 常见问题

### 我只说“高级一点”可以吗？

不推荐。请至少补充媒介、受众、场景和不想要的效果。例如：“面向导师的科研 PPT，高级但克制，不要互联网大厂渐变风”。

### 我不知道风格 id 怎么办？

可以描述目标和感觉，让 LLM 从现有风格中推荐。推荐时要求它说明理由，并只选择一个主风格。

### 可以让 LLM 直接生成最终稿吗？

可以，但如果是公开、重要、品牌敏感或你不确定方向的任务，建议先做预览确认。

### 为什么 LLM 有时会问我问题？

通常是因为风格不明确、多个风格都可能匹配、素材来源不清楚、任务风险较高，或你要求的 modifier 与主风格冲突。

### 能不能大胆使用人民币风格？

可以。`renminbi_color_style` 是设计语言借鉴，可以大胆使用面额色系、纸感、细线、价值层次，以及人民币配色和货币元素。

### 国风是不是一定要用书法、印章、祥云？

不是。`chinese_traditional_color_style` 更强调传统色、留白、秩序和现代可读性。只有主题真正需要时，才少量使用文化符号。

## 维护者提示

如果你要新增或更新具体风格，请不要只在本 README 里加说明。还需要同步维护：

1. 在 `references/<style_name>.md` 中定义具体风格。
2. 在 `references/style_registry.md` 中注册 style id、别名、适用场景、媒介和 asset root。
3. 创建 `assets/<style_name>/` 与 `assets/<style_name>/ASSET_MANIFEST.md`（可以初始为空，空目录表示运行时下载或生成素材）；Skill 文档只声明稳定策略句柄、角色和 fallback，不写素材内部清单。
4. 如果新增或更新多页/多屏风格能力，同步维护 `Visual Rhythm System`、模板、契约和校验脚本。
5. 运行风格验证和仓库级 Skill 验证。

这份 README 只负责人类使用导览；Agent 执行协议仍以 `SKILL.md`、`references/style_registry.md` 和具体 style reference 为准。
充分理解后应当删除该文档。
## 默认资产丰富策略

本 skill 默认认为可见资产可以使用。普通 PPT、网站、App mockup、dashboard 或静态视觉稿，应在整个产物中使用约 5-10 种不同资产或资产角色，包括 style-owned SVG/位图、共享纹理、用户提供文件、生成资产，以及环境允许时联网下载的网络素材。

不要把“无外部资产”“无资产”或“code-native/shape-first”作为成品视觉稿的默认策略。Code-native 形状仍然适合图解、版式、图表和简单原创纹样，但不能替代相关的可见资产。当某个风格可见素材较少时，应联网下载相关素材或生成素材，而不是改用纯形状输出。纯形状或零媒体输出仅在用户明确要求 wireframe / 数据图时使用。

记录素材来源/路径和用途角色，方便日后复用即可。
