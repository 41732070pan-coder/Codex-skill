# Learning Strategy Contract

Use this reference to decide how much attention each knowledge point deserves and how to check retention.

## Depth Decision

Score each knowledge point from 1 to 5 on these axes:

| Axis | Meaning |
| --- | --- |
| Importance | How central the point is to the tutorial's main skill. |
| Prerequisite value | Whether later sections depend on it. |
| Practical frequency | How often the learner will use it in real work. |
| Complexity | How much explanation and practice it needs. |
| Goal relevance | How strongly it matches the user's stated goal. |

Use this policy:

| Selected depth | Rule of thumb | Lesson treatment |
| --- | --- | --- |
| `deep` | High importance or prerequisite value and medium/high goal relevance. | Full explanation, example, practice, checkpoint, and review card. |
| `normal` | Useful but not foundational, or foundational but easy. | Concise explanation, one check, one light practice. |
| `skim` | Helpful orientation but low practical frequency or low goal relevance. | Bullet summary and optional check. |
| `skip_with_link` | Complex, rare, repetitive, or outside the current goal. | State why it is skipped and provide a deep-dive target if useful. |

## Time Budgeting

- Default lesson size: 15-45 minutes.
- Prefer several short lessons over one overloaded lesson.
- If time is constrained, keep `deep` items, compress `normal` items, skim reading-only content, and skip low-value complex details.
- If the learner is advanced, reduce definitions and add transfer tasks.
- If the learner is novice, add prerequisites and smaller checkpoints.

## Practice Policy

| Knowledge profile | Practice type |
| --- | --- |
| Conceptual but not procedural | Explain-in-your-own-words, compare/contrast, mini scenario. |
| Procedural | Follow steps, then modify one condition. |
| Code/tool based | Run or mentally simulate a small example. |
| Diagnostic | Given symptoms, identify cause or next action. |
| Advanced optional | Recognition question only unless user requests depth. |

## Review Policy

Every lesson should include:

- immediate checkpoint questions;
- one next-day review prompt;
- one later review prompt;
- a mastery criterion expressed as an observable action.
