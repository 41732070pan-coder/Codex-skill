# Web Course Workflow

Use this workflow for online tutorials, documentation tutorials, course pages, or article series.

## Workflow

1. Capture source title, URL, publisher/author when visible, access date, and selected page or section.
2. Separate tutorial body from navigation, sidebars, comments, ads, cookie notices, marketing copy, and unrelated links.
3. Map headings and next/previous links into a chapter or section sequence.
4. Preserve important source links in lesson metadata or further reading.
5. Classify content into core knowledge, supporting examples, reading-only context, skip candidates, and advanced optional material.
6. Generate a markdown lesson with checkpoints and review prompts.
7. Keep attribution and source URLs in the artifact.

## Web-Specific Heuristics

| Signal | Interpretation |
| --- | --- |
| Main article heading hierarchy | Primary section structure. |
| Navigation sidebar | Course map, not lesson content unless the user asks for a syllabus. |
| API reference blocks | Advanced optional unless the tutorial section teaches that API directly. |
| Marketing or product positioning | Reading-only or skip candidate. |
| Comments and social widgets | Skip. |
| External docs link | Candidate deep-dive or prerequisite link. |
