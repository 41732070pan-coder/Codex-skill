# Iteration Loop Contract

Use this contract when refining the skill through repeated source-to-lecture experiments.

## Loop Shape

Each round has four steps:

1. Update the high-level abstraction when a contract gap appears.
2. Update the PDF implementation when a section parsing, triage, or output rule needs improvement.
3. Regenerate the first-section lecture for the current PDF source.
4. Evaluate the skill and lecture with a rubric, then record a concrete change request.

## Required Evidence

- The same PDF remains fixed for rounds 1-5.
- A different PDF is selected for rounds 6-10.
- Every round records what changed and why.
- Every round records at least one rubric-based improvement.
- The log must show both source selection and evaluation closure.

## Switch Rule

Use the first PDF to stabilize the abstraction and the chaptered-PDF implementation. Switch to a second public tutorial PDF after round 5 to test whether the skill still works when the topic changes.
